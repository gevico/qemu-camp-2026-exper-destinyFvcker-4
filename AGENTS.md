# AGENTS.md

## 项目定位

- 本项目是一个以 `QEMU` 为主线的学习性项目，重点是理解 `QEMU`、`RISC-V virt`、启动流程、设备模型、调试方法和相关源码。
- 在本项目中工作时，优先服务“学习、解释、沉淀知识”，而不是把它当成产品型仓库来处理。

## 问答后的 `learning/` 沉淀规则

- 智能体在每次问答结束后，都需要额外判断：这次回答的内容是否值得沉淀到 `learning/` 目录。
- 如果内容具有长期复用价值、能帮助后续理解项目，或未来大概率还会再次用到，就应该记录下来。
- 典型值得记录的内容包括：
  - 对 `QEMU` / `RISC-V` / 启动流程 / 设备模型 / 调试流程 / 构建流程的稳定解释
  - 源码阅读路径、关键文件定位、关键函数关系
  - 常用命令、排错结论、概念辨析、术语解释
- 通常不必记录的内容包括：
  - 纯闲聊或一次性沟通
  - 与本项目学习目标无关的泛化问题
  - 还没有形成稳定结论、明显属于猜测的内容

## 记录方式

- 在写入 `learning/` 之前，先检查已有的 `.md` 文件，优先合并到最相关的现有文档中。
- 只有在现有文档都明显不适合承载该内容时，才新建 `.md` 文件。
- 记录内容时，应整理成便于后续复习的学习笔记，而不是机械复制整段问答。
- 如果新开文件，尽量在相关的已有文档中补充链接，避免 `learning/` 目录碎片化。
- 如无特殊说明，笔记使用中文；缩写第一次出现时尽量补全英文全称和中文含义。

## 表达方式偏好

- 面向初学者解释抽象概念、层级关系、对象关系、初始化流程时，优先先给“整体图”再补文字说明。
- 这类内容尽量整理成关系图、流程图、时序图等可复习的结构化材料。
- 可以使用 `Mermaid` 画图；如果 `Mermaid` 能更清楚表达关系，就优先使用。

## 回答后的说明

- 如果本次问答已被沉淀到 `learning/`，请在最终回复中明确说明更新了哪个文件。
- 如果判断这次内容不值得沉淀，也请在最终回复中简要说明“这次未记录到 `learning/`”。
